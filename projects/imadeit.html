About the Project

ImadeIt is a full-stack showcase platform built for makers across every discipline — anyone who wants a place to publish the work they've made and the story behind it. The interface borrows from two different worlds: an Instagram-style grid for browsing visual work at a glance, and a Medium-style detail view for the long-form write-up of materials, process, decisions, and lessons that produced each piece.

Creators sign in with a magic link — no passwords — and a database trigger provisions a profile row automatically on first sign-in. From their dashboard they can create projects with a cover image, a Markdown body, and inline images that can be uploaded mid-write and inserted at the cursor. Each project is independently set to Published, Unlisted, or Draft, comments can be toggled per project, and a slide-in Markdown cheat-sheet offers click-to-insert snippets that wrap the current selection. Browsers get public feeds, per-category pages over a curated 18-tag taxonomy, per-user portfolio pages, threaded comments, and a daily-deduplicated view counter.

The defining architectural choice is that authorization lives in the database, not the app. Every public table has Row Level Security enabled, with policies that constrain reads (published / unlisted only; drafts author-only) and writes to the row's owner. Comment-disabling is enforced at the policy layer, so even a hostile client crafting requests directly against the Supabase REST endpoint can't bypass it. View tracking flows through a SECURITY DEFINER Postgres function with an ON CONFLICT DO NOTHING insert keyed on (post_id, viewer_hash, viewed_on), making it both forgery-resistant and naturally idempotent. Storage uploads are gated by path so the bucket policy permits inserts only when the object key starts with the user's UID. Schema is owned by Drizzle as the single source of truth, with SQL migrations applied by an idempotent runner that tracks applied files in a _migrations ledger table.
